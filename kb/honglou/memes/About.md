## 构建方式

知识库由 **Wiki Butler**（AI 自动化[[管家]]）持续维护：

- **原文**：[[程甲本]]一百二十回全文，每段标注 PN（段落编号），可精确定位
- **知识图谱**：从原文提取的实体、事件、关系，支持语义 wikilink（`[[rel::target]]`）
- **精品页**：重要人物（[[贾宝玉]]、[[林黛玉]]、[[王熙凤]]等）经过深化，引用原文段落号和他人评价
- **内容质量**：每页有自动计算的 K 分，驱动首页排序

所有页面编辑留有修订历史，可通过 [[Special:Recent|最近修订]] 查看变更记录。

## 方法论

### LLM Agent 驱动的知识构造

本 Wiki 的全部内容由 **Claude Code Agent（Wiki Butler）** 自动化构造：

1. **原文解析**：Agent 逐章阅读[[程甲本]]原文，标注段落编号（PN），建立精确的原文引用体系
2. **实体发现**：从原文中自动识别人物、地点、[[器物]]、事件、概念等实体，补全缺失词条
3. **关系提取**：分析实体间的亲属、[[主仆关系|主仆]]、空间、[[因果]]等关系，生成结构化元数据
4. **质量迭代**：通过 enrich 流水线持续提升词条质量（stub。 basic。 standard），补充 PN 引文和交叉链接
5. **语义推理**：利用 `[[rel::target]]` 语义 wikilink 构建推理知识库，支持关系查询

### Semantic Wiki — 可查询的页面元数据

每个词条的 **YAML frontmatter** 构成结构化元数据层，可被浏览器端搜索引擎和查询界面利用：

- `type`：页面类型分类，支持按类型筛选（`#?type=person`）
- `aliases`：别名索引，搜索别名即可定位到目标词条
- `tags`：自由标签体系，支持多维度分类
- `quality` + `quality_score`：内容质量评级，驱动首页排序
- 类型专属字段：人物有 `master`（所属主⼈）/`fate`（[[人物结局|结局]]），地点有 `所属`，事件有 `时间`/`地点` 等

注册表（`pages.json`）汇总全部页面的元数据，浏览器端 JavaScript 可在不加载全文的情况下执行跨页查询、统计和筛选。你也可以在 URL 上直接使用 `#?type=person&tag=[[荣国府]]` 等参数进行过滤。

## 如何使用

- **搜索**：在首页搜索框输入人名、别名（如「[[林黛玉|颦颦]]」「[[怡红公子]]」均可找到[[贾宝玉]]）；可切换到全文检索模式搜索原文
- **浏览**：[[Special:AllPages]] 查看全部页面，支持按类型、标签筛选
- **随机**：[[Special:Random]] 随机跳转一页
- **最近更新**：[[Special:Recent|最近修订]] 查看近期变化
- **知识量**：[[Special:Statistics]] 查看知识库统计指标
