## BusyBeaver是什么？

BusyBeaver是一个由LLM构建的知识库，其使命是把对人类最重要的知识——主要来自经典著作——转化为一个**对agent友好**的知识库。

我们的做法是把知识拆解成小的、可分发的、语义丰富的片段。我们把每个片段称为一个**Meme**。

> **Meme**一词由英国生物学家**Richard Dawkins**在《自私的基因》(The Selfish Gene)（1976年）中首创，他将其定义为**文化传播的基本单位**。BusyBeaver把这个概念用得很直白：一个Meme就是一个足够小、能够传播、复制、重组的知识单元。

- 每个**Meme**都携带自己的**元数据**。
- 每个**Meme**都可以被**查询**。
- **Memes**可以**自由组合**成wiki、简报或答案。

本仓库是一份**dump（数据转储）**：数百本经典著作经memify（转化为Meme）后的拆解结果。这个语料库正在**快速增长**——我们的第一个里程碑是**10,000本书**。

> **为什么叫"Busy Beaver"？** 在可计算性理论中，**Busy Beaver函数**是*增长最快*函数的典型代表——它超越一切可计算函数的增长速度。这正是这个名字背后的雄心：为即将到来的agent世界打造一个**超级知识库**，以我们能达到的最快速度增长。

如果你把一个agent指向BusyBeaver，它消化经典著作的速度会*显著加快*，因为最难的工作——拆分文本、分析文本、把文本关联起来——已经完成了。

## 为什么存在

我们想要延续**语义网**（Semantic Web）与**知识图谱**（Knowledge Graphs）未竟的承诺，同时摆脱经典知识表示技术栈的局限。

关键的转变在于：**知识的消费者现在是一个agent。**

正因如此，知识不再必须被编码成RDF三元组或形式逻辑表达式。它可以变成agent真正最擅长消费的形式——**语义丰富的自然语言句子，配以元数据**，切分成agent可以自由拆分、重组、推理的片段。

> 经典的知识表示方法把人类的含义强行塞进狭窄的形式化语法中。
> BusyBeaver把含义留在语言里，让结构存在于元数据中。

BusyBeaver站在此前伟大的结构化知识项目的肩膀上。我们尤其要感谢：

- **[Freebase](https://en.wikipedia.org/wiki/Freebase_(database))** — 一个开放的实体与关系图谱。
- **[Semantic MediaWiki](https://www.semantic-mediawiki.org/)** — 把语义结构带入wiki本身。

为了构建BusyBeaver，我们设计了一套属于自己的**全新语义wiki方法**——在这套方法中，知识的单位是一个可溯源到源头、agent可消费的Meme，而不是一个僵硬的三元组。这套方法将会**单独开源**。

## 最难的问题：幻觉与溯源

用LLM构建知识库最难的一部分，是**检测和控制幻觉**。

BusyBeaver的答案是**溯源控制**。我们尽可能忠实地保留知识的*出处*，使得**每一条论断都能追溯回它的来源**——通常是某本书中的具体段落。

每个知识来源都被赋予一个**PN——Purple Number**。

> Purple Number体现的是**Doug Engelbart的理想**：*世界上的每一条知识都应该拥有一个地址。*

有了PN，BusyBeaver中的一条论断永远不是一句悬空的陈述。它是被锚定的——可寻址、可核查、可引用回其来源。

## Meme的解剖结构

Meme是BusyBeaver的原子单位。从概念上讲，每个Meme捆绑了：

| 部分 | 描述 |
| --- | --- |
| **Assertion（论断）** | 一条自成体系、语义丰富的自然语言知识陈述。 |
| **Metadata（元数据）** | 类型、主题、实体、关系，以及其他用于检索与组合的结构化信号。 |
| **Provenance (PN)（溯源）** | 一个把论断锚定到其源段落的Purple Number，使论断保持可溯源。 |

由于每个Meme都很小、能自我描述、且可溯源到来源，Memes既是**可分发的**（分享一个而无需运送整本书），也是**可组合的**（把多个组装成一个wiki页面、一份简报，或一个答案）。

## 📦 知识的包交换

全球基础设施的每一次飞跃，都源于一种新的**交换单元**：

- **数据的包交换**把信息拆成可路由的小数据包——催生了**互联网**。
- **集装箱运输**把货物装进标准、可互换的箱子——推动了**贸易全球化**。

**Meme 就是知识的这个交换单元**：一个小巧、自描述、锚定源头的知识包，可在不同系统与组织之间被路由、交换、聚合与重组。让**知识的包交换**真正跑起来，正是我们建立**全球知识基础设施**的方式。

## 为什么要用知识库，而不只是原始文本？

如果agent能读，为什么不直接把书喂给它？因为BusyBeaver提供了两样原始文本无法提供的东西：

### 1. 一套控制幻觉的溯源框架
把原始文本再喂回LLM，会重新打开LLM知识库最大的缺陷之门：**幻觉**。BusyBeaver把每一条论断都包裹在**溯源追踪**之中——每个论断都通过其PN锚定到它来自的源段落。这给了你一个框架去*核实*agent告诉你的内容，而不是全凭信任。

### 2. 一套现成的语义骨架
我们已经完成了系统性的**拆解**工作，并把它**关联回原始文本**。散文中蕴藏的关键语义结构已经被提炼成**概念（concepts）与属性（properties）**——一套你可以立即在其上构建的语义骨架。这比从原始文本出发、自己重建这一切要省力得多。

> 原始文本给你的agent的是*可以阅读的文字*。
> BusyBeaver给它的是*可以信任、且已经结构化并相互关联的知识*。

### 🌿 草 vs. 🥩 肉

不妨这样想：**原始文本是草；BusyBeaver知识库是肉。**

你*可以*直接吃草——但那样你就需要一套庞大且昂贵的消化系统去分解它。改吃肉的话，你就省下了本该花在消化草上的所有能量——对你的agent来说，这份能量就是**LLM tokens**——转而把它花在更聪明的事情上。

**肉是高密度的知识：高度压缩的上下文。** BusyBeaver已经替你完成了消化，你的agent就不必再做了。

## 如何使用

BusyBeaver **不强制要求任何特定的存储或检索方式**。每一段文本、每一条衍生知识都已经被仔细切分、拆解、分析——所以你可以把它接入你已经在用的任何技术栈：

### 🔎 向量存储 / RAG
索引这些Memes并按语义相似度检索。这些片段的大小与清洗都已经为embedding做好准备——不需要额外的分块流程。

### 🗂️ 经典搜索（Elasticsearch及同类工具）
把Memes及其元数据载入Elasticsearch，或任何传统检索引擎，按字段、分面（facets）与全文进行查询。

### 🤖 编程agent（例如Claude Code）
把像**Claude Code**这样的编程agent直接指向本仓库，让它直接读取这个知识库——让它在整个语料库中进行分析、交叉引用、得出结论。

无论你选择哪条路径，PN溯源都会随知识一同流动，使你的agent得出的结论始终**可审计**。

## 🛠️ 你能用 BusyBeaver 做什么？

- **进行知识的二次创造。** 把这些知识当作素材，创作你的博客、PPT、小说、剧本、论文、漫画、视频等等。
- **为你的 agent 提供选定主题的上下文。** 喂给它有选择的主题，让它成为某件事情的专家。
- **训练你的模型。** 在结构化的 Meme 上训练——比直接读原始文本更高效、更省 token。
- **增长出你自己的 KB。** 把 BusyBeaver 当作基础知识层，去孵化和扩展你自己的知识库（比如你的 wiki）。
- **做整合，制造 serendipity。** 因为这里的知识是可查询的，你可以做基于知识的数据整合——用 query 创造 serendipity，发现你原本不知道该去找的关联。

### 🌍 一个多语言知识库

BusyBeaver的构建目标是**国际化**。初始语料库主要以**英文和中文**为主，我们会随着时间推移持续加入世界主要语言——因为塑造人类文明的经典著作并非只用一种语言写成。

### 🔮 即将到来

BusyBeaver本身是语料库。围绕它，我们正在构建：

- **每本书的交互式wiki** — 把每本书的知识作为一个鲜活的、相互链接的wiki来浏览与查询，而不只是一份扁平的dump。
- **任意内容 → wiki** — 一项能把*任意*内容转化为结构化、面向agent的wiki的服务——包括**你自己拥有版权的资料**（内部手册、授权文本、专有研究），让你拥有一个属于自己的私有知识库。
- **联邦式语义wiki** — 一套共享协议，让知识agent能够跨独立的wiki**自由交换、聚合、查询**知识。一个地址空间，多方参与者——Purple-Number的梦想，以联邦形式实现。

## 🚀 你的知识，也能 memify

BusyBeaver 只是开始。我们计划帮**你**把*你自己的*知识——**个人笔记与企业知识皆可**——变成同样的 KB：**可分发、可组合、可被 agent 轻松消费。**

经典只是我们的起点。更大的愿景，是一个开放、联邦化、可信的 agent 原生知识经济——而你的知识，在其中占有一席之地。
